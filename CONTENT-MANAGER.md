# Lunar-Tear Content Manager (Dynamic Master Data)

This document provides an in-depth technical overview of the new configuration-driven master data scheduling system implemented in `lunar-tear` to resolve severe UI lag, memory bloat, and content overload.

## Architecture & Problem Statement
Running the final EOS (End of Service) `database.bin.e` directly causes the Nier Re[in]carnation client to attempt to render over 140+ side stories and 80+ Gacha banners natively in the Subquests and Summon UI. This causes massive framerate drops and menu freezing because the client was never designed for this load.

Instead of statically serving the bloated EOS database, this PR introduces a dynamic patching pipeline. The `lunar-tear` Go server tracks a JSON-based schedule state and dynamically invokes a localized Python script that un-encrypts, mutates, and re-encrypts the `database.bin.e` SQLite blob on the fly, perfectly syncing client content with the admin configuration.

## Technical Implementation Breakdown

### 1. The Binary Patcher (`lunar-scripts/patch_masterdata.py`)
This is the core workhorse of the new system. We integrated a new `--sync-schedule` command line flag that dynamically filters content.
* **MessagePack Stream Manipulation:** The Nier Reincarnation SQLite database stores actual table rows as deeply nested, LZ4-compressed `ExtType(99)` MessagePack blobs. The Python script parses these arrays in memory.
* **Selective Row Disablement:** It reads `content_schedule.json` generated by the Go admin panel, matches it against `bundle_index.json`, and selectively loads active `event_chapters`, `gacha_ids`, and `login_bonuses`. 
* **In-Place Offset Patching:** The script iterates through the msgpack array without full deserialization. It looks at the index `0` integer (e.g., `EventQuestChapterId` or `MomBannerId`). If the ID is *not* in the active schedule constraints, the `EndDatetime` `int64` timestamp is patched to `2020-01-01` (flagging it as organically expired to the client). If it *is* in the constraints, it's bumped safely to `2030-12-31`.
* **Zero-Banner Failsafe:** The Nier client possesses a known edge-case where opening the 3D Gacha room with exactly zero active banners triggers an unhandled `IndexOutOfRangeException` and freezes the game. To prevent this, if the python script detects `allowed_gacha` is empty, it automatically injects a permanent fallback (IDs 45 & 46 - Automata Crossover) into the byte stream.

### 2. The Go Schedule Manager (`server/internal/schedule/manager.go`)
This package was entirely abstracted to manage the synchronization lifecycle.
* **State Orchestration:** When a user updates the Admin UI or clicks a Preset, the manager commits the new state to `content_schedule.json`.
* **Synchronous Invader:** It natively invokes `exec.Command` on `patch_masterdata.py`, routing standard output for debugging. It guarantees that any UI changes force an immediate rewrite of the `assets/release/database.bin.e` output file before returning HTTP 200 to the UI.
* **Timestamp Tracking:** Generates a thread-safe atomic integer metric tracking `lastUpdatedMillis`.

### 3. Dynamic Cache Invalidation (`server/internal/service/data.go`)
Historically, updating the Master Data required clearing the app cache completely because the mobile client caches `database.bin.e` heavily and doesn't verify byte sizes sequentially.
* We intercepted the `GetLatestMasterDataVersion` gRPC endpoint in `DataServiceServer`.
* Instead of returning the static EOS string `"20240404193219"`, we append the Schedule Manager's state timestamp as a suffix (e.g., `20240404193219_1713838290`).
* **Result:** Returning to the Title Screen forces the mobile client to realize its cache string mismatches the payload string, forcing it to seamlessly redownload the lightweight 7MB difference automatically in under a second.

### 4. Admin UI Overhaul (`server/internal/service/admin_ui.go`)
* **Bug Fixes:** A state re-render bug where pressing "All Content" triggered a page reload and cleared inputs was resolved. The bundles are now cached locally via Javascript on initial load, preventing infinite fetch loops whenever a preset is clicked.
* **Real-time API Binding:** Clicking "Apply" dumps an instant `POST` payload directly mapping `schedule.active_bundles` strings cleanly to the Go HTTP server.

## Important Historical Context for Reviewers
If you notice that some tabs or older events behave differently from release, this is an effect of the EOS database itself, not the patcher.
* **Missing Limited Event Banners:** Over the course of the game's final year, Applibot largely deprecated the Subquest "Events" tab and migrated all limited record/collaboration crossovers permanently to the **"Memorial Quests"** category as part of the EOS transition. The database no longer stores the UI mapping required to show them as limited banners. They trigger, but unlock sequentially in the memorial section instead.
* **Stories vs. Events:** The Admin UI distinguishes "Stories" (Recollections of Dusk) via `m_side_story_quest_limit_content`. Because that specific UI system was introduced to the game only in **August 2022**, there will be technically zero "Stories" available when simulating bundles from 2021. This is perfectly accurate representation of the historic client's restrictions.
